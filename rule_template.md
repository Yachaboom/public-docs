이 프로젝트에는 {{ sdk_name }} {{ version }}의 완전한 참조 문서가 포함되어 있습니다. 아래 3개 파일을 활용하여 {{ sdk_name }} 관련 질문에 정확하고 실용적인 답변을 제공하세요.

## 문서 파일 구조

### 1. `main-documentation.md`

**완전한 참조 문서** - 가장 상세하고 포괄적인 문서

- 설치부터 프로덕션 배포까지 전체 과정
- 실행 가능한 코드 예제 포함
- 다수의 주요 섹션으로 구성
- 특정 주제에 대한 깊이 있는 설명이 필요할 때 참조

### 2. `llms.txt`

**빠른 참조용 인덱스** - 문서 구조와 핵심 개념 요약

- 전체 문서 구조 파악
- 특정 섹션 빠르게 찾기
- 개념적 이해를 위한 첫 번째 참조점
- 토큰 사용량 최소화

### 3. `llms-full.txt`

**통합 전체 내용** - 모든 내용을 하나의 파일로 통합

- 전체 맥락이 필요한 복잡한 질문
- 여러 섹션에 걸친 정보 통합
- 완전한 컨텍스트 검색
- 복합적인 구현 요구사항 처리

### 질문 유형별 문서 활용 전략

#### 1. 빠른 정보 확인 (Simple Query)
```
질문 예시: 
- "{{ sdk_name }}에서 [특정 기능] 사용법이 뭐야?"
- "[API/메서드명] 매개변수가 어떻게 되나?"
- "[설정옵션] 기본값이 뭐야?"

접근법: llms.txt → main-documentation.md 해당 섹션
```

#### 2. 구현 예제 필요 (Implementation Example)
```
질문 예시:
- "[특정 기능]을 구현하려면 어떻게 해야 해?"
- "[사용 사례] 완전한 코드 예제를 보여줘"
- "[문제 상황] 해결하는 방법 알려줘"

접근법: main-documentation.md의 "통합 패턴과 실전 예제" 섹션
```

#### 3. 전체적인 이해 필요 (Conceptual Understanding)
```
질문 예시:
- "{{ sdk_name }} 아키텍처를 전반적으로 설명해줘"
- "[핵심 개념] 동작 원리가 뭐야?"
- "{{ sdk_name }}과 [다른 기술] 차이점이 뭐야?"

접근법: llms-full.txt 또는 main-documentation.md의 "핵심 개념" 섹션
```

#### 4. 복잡한 통합 질문 (Complex Integration)
```
질문 예시:
- "프로덕션 환경에서 성능 최적화하면서 에러 처리까지 고려한 [시스템]을 만들려면?"
- "[기술A]와 [기술B]를 통합해서 [복잡한 기능] 구현하려면?"
- "[전체 시스템] 아키텍처 설계 가이드를 제공해줘"

접근법: llms-full.txt에서 여러 섹션 통합 참조
```

